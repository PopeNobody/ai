Erläuterung der API-Versionen

Mit Sammlungen den Überblick behalten Sie können Inhalte basierend auf Ihren Einstellungen speichern und kategorisieren.

Dieses Dokument bietet einen allgemeinen Überblick über die Unterschiede zwischen Version 1 und Version 1. und v1beta der Gemini API.

  • v1: stabile Version der API Die Funktionen der stabilen Version während der gesamten Lebensdauer der Hauptversion vollständig unterstützt wird. Falls es welche gibt, funktionsgefährdende Änderungen vornehmen, wird die nächste Hauptversion der API erstellt wird die vorhandene Version nach einem angemessenen Zeitraum verworfen. Abwärtskompatible Änderungen an der API können ohne Änderung der wichtigsten Version.
  • v1beta: Diese Version enthält Early-Access-Funktionen, die möglicherweise und unterliegt schnellen und funktionsgefährdenden Änderungen. Es gibt auch keine Garantie dafür, dass die Funktionen in der Betaversion in die stabile Version übertragen werden Version. Aufgrund dieser Instabilität solltest du die Produktionsversion nicht starten Anwendungen mit dieser Version.
