Afficher sur ai.google.dev Exécuter dans Google Colab Consulter le code source sur GitHub

Si vous souhaitez essayer rapidement l'API Gemini, vous pouvez exécuter des commandes curl pour appeler les méthodes dans l'API REST. Les exemples de ce le tutoriel affiche les appels pour chaque méthode API.

La Colab utilise du code Python pour définir une variable d'environnement et afficher une image, mais vous n'ont pas besoin de Colab pour fonctionner avec l'API REST. Vous devriez être en mesure d’exécuter toutes les curl exemples en dehors de Colab, sans modification, à condition que vous ayez API_KEY défini comme décrit dans la section suivante.

Pour chaque commande curl, vous devez spécifier le nom du modèle applicable et votre API. .

Configurer votre clé API

Pour utiliser l'API Gemini, vous avez besoin d'une clé API. Si vous ne l'avez pas encore fait, créez une clé dans Google AI Studio.

Obtenir une clé API

Dans Colab, ajoutez la clé au gestionnaire de secrets sous l'icône " succès" dans le panneau de gauche. Nommez-la GOOGLE_API_KEY. Vous pouvez ensuite l'ajouter en tant que variable d'environnement pour transmettre la clé dans votre appel curl.

Dans un terminal, vous pouvez simplement exécuter GOOGLE_API_KEY="Your API Key".

import os
from google.colab import userdata

os.environ['GOOGLE_API_KEY'] = userdata.get('GOOGLE_API_KEY')

Saisie de texte et d'image

Gemini fournit différents modèles pouvant gérer la saisie multimodale (modèles Gemini 1.5) afin que vous puissiez saisir à la fois du texte et des images. N'oubliez pas de consulter les exigences concernant les images pour les requêtes.

Lorsque la requête inclut à la fois du texte et des images, utilisez un modèle Gemini 1.5.

Les extraits de code suivants vous aident à créer une requête et à l'envoyer à l'API REST.

curl -o image.jpg https://storage.googleapis.com/generativeai-downloads/images/scones.jpg

Conversations multitours (chat)

Avec Gemini, vous pouvez construire des conversations libres dans plusieurs tours.

curl https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=$GOOGLE_API_KEY \
    -H 'Content-Type: application/json' \
    -X POST \
    -d '{
      "contents": [
        {"role":"user",
         "parts":[{
           "text": "Write the first line of a story about a magic backpack."}]},
        {"role": "model",

Configuration

Chaque requête que vous envoyez au modèle inclut des valeurs de paramètre qui contrôlent la manière dont le modèle génère sa réponse. Le modèle peut générer différents résultats pour différentes valeurs de paramètre. En savoir plus sur les paramètres de modèle

Vous pouvez également utiliser les paramètres de sécurité pour ajuster la probabilité d'obtenir des réponses susceptibles d'être considérées comme nuisibles. Par défaut, les paramètres de sécurité bloquent le contenu dont la probabilité est moyenne et/ou élevée d'être dangereux pour toutes les dimensions. En savoir plus sur les paramètres de sécurité

L'exemple suivant spécifie des valeurs pour tous les paramètres de la méthode generateContent.

Embedding

La représentation vectorielle continue est une technique utilisée pour représenter des informations sous la forme d'une liste de nombres à virgule flottante dans un tableau. Avec Gemini, vous pouvez représenter du texte (mots, phrases et blocs de texte) sous forme vectorisée, ce qui facilite la comparaison des représentations vectorielles continues. Par exemple, deux textes qui partagent un sujet ou un sentiment similaire doivent avoir des représentations vectorielles continues similaires, qui peuvent être identifiées à l'aide de techniques de comparaison mathématique telles que la similarité cosinus.

Utilisez le modèle embedding-001 avec embedContents ou batchEmbedContents:
