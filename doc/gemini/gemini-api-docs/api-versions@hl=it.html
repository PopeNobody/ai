Spiegazione delle versioni API

Mantieni tutto organizzato con le raccolte Salva e classifica i contenuti in base alle tue preferenze.

Questo documento fornisce una panoramica di alto livello delle differenze tra la versione 1 e e le versioni v1beta dell'API Gemini.

  • v1: versione stabile dell'API. Le funzionalità nella versione stabile sono completamente supportata per tutta la durata della versione principale. Se esistono le modifiche che provocano un errore, verrà creata la successiva versione principale dell'API la versione esistente verrà deprecata dopo un ragionevole periodo di tempo. È possibile introdurre modifiche continue all'API senza modificare i principali completamente gestita.
  • v1beta: questa versione include funzionalità di accesso in anteprima che potrebbero essere in sviluppo ed è soggetto a rapidi cambiamenti. Inoltre, non c'è garantire che le funzionalità della versione beta vengano trasferite alla versione stabile completamente gestita. A causa di questa instabilità, non dovresti avviare la produzione applicazioni con questa versione.
