Omówienie wersji interfejsu API

Zadbaj o dobrą organizację dzięki kolekcji Zapisuj i kategoryzuj treści zgodnie ze swoimi preferencjami.

Ten dokument zawiera ogólny przegląd różnic między wersją 1 Gemini API, jak i wersje v1beta.

  • v1: stabilna wersja interfejsu API. Funkcje w wersji stabilnej to: w pełni obsługiwane przez cały okres istnienia wersji głównej. Jeśli są zmian powodujących niezgodność, zostanie utworzona następna główna wersja API bieżąca wersja zostanie po pewnym czasie wycofana. Można wprowadzić do interfejsu API nienaruszające zmiany bez zmiany głównych wersji.
  • v1beta: ta wersja zawiera funkcje wcześniejszego dostępu, które mogą być na poziomie szybko się rozwija i podlega nagłym zmianom. Nie ma także że funkcje z wersji beta zostaną przeniesione do stabilnej wersji wersji. Ze względu na niestabilność nie należy uruchamiać wersji produkcyjnej aplikacji w tej wersji.
