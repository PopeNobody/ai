הוספת דוגמאות של 'ללא הרבה דוגמאות'

אפשר לכלול דוגמאות בהנחיה שמראות איך המודל עובד כמו שצריך. המודל מנסה לזהות דפוסים ויחסים מהדוגמאות ולהחיל אותם בזמן יצירת התשובה. הנחיות שמכילות כמה דוגמאות נקראות הנחיות ל-few-shot, ואילו הנחיות שלא כוללות דוגמאות נקראות הנחיות ל-zero-shot. הנחיות ב-Few-shot משמשות לרוב לוויסות הפורמט, הניסוח, ההיקף או התבנית הכללית של התשובות של המודל. כדאי להשתמש בדוגמאות ספציפיות ומגוונות כדי לעזור למודל לצמצם את המיקוד שלו ולהשיג תוצאות מדויקות יותר.

תמיד מומלץ לכלול בהנחיות את הדוגמאות של כמה דוגמאות. סביר להניח שהנחיות ללא דוגמאות של 'כמה תמונות בלבד' יהיו פחות יעילות. למעשה, אפשר להסיר את ההוראות מההנחיה אם הדוגמאות מספיק ברורות כדי להראות את המשימה.

שימוש בפורמט עקבי בכל הדוגמאות

  • כדאי להתנסות במספר ההנחיות להכללה. בהתאם למודל, מעט מדי דוגמאות לא יעילות בשינוי ההתנהגות של המודל. יותר מדי דוגמאות גורמות למודל להתאים את עצמו יותר מדי לנתונים.

מודלים גנרטיביים של שפה פועלים כמו כלי מתקדם להשלמה אוטומטית. כשאתם מספקים תוכן חלקי, המודל יכול לספק את שאר התוכן או את מה שהוא חושב שהוא המשך של התוכן הזה בתור תשובה. כשאתם עושים זאת, אם תכללו דוגמאות או הקשר, המודל יוכל להביא בחשבון את הדוגמאות או ההקשר האלה.

בדוגמה הבאה מוצגת הנחיה עם קלט של ישות:

For the given order, return a JSON object that has the fields cheeseburger, hamburger, fries, or
drink, with the value being the quantity.

Order: A burger and a drink.
  
```
{
  "hamburger": 1,
  "drink": 1
}
```

המודל אכן פעל לפי ההנחיה, אבל לפעמים קשה לכתוב את ההוראות בשפה טבעית, והרבה תלוי בפרשנות של המודל. לדוגמה, תפריט של מסעדה יכול להכיל הרבה פריטים. כדי לצמצם את גודל התגובה של ה-JSON, מומלץ להשמיט את הפריטים שלא הוזמנו. במקרה כזה, אפשר לתת דוגמה וקידומת לתשובה ולאפשר למודל להשלים אותה:

Valid fields are cheeseburger, hamburger, fries, and drink.
Order: Give me a cheeseburger and fries
Output:
```
{
  "cheeseburger": 1,
  "fries": 1
}
```
Order: I want two burgers, a drink, and fries.
Output:
  
```
{
  "hamburger": 2,
  "drink": 1,
  "fries": 1
```

Create an outline for an essay about hummingbirds.

סיכום

  • אם נותנים למודל קלט חלקי, הוא משלים את הקלט הזה על סמך דוגמאות או הקשר זמינים בהנחיה.
  • לפעמים קל יותר לבקש מהמודל להשלים קלט מאשר לתאר את המשימה בשפה טבעית.
  • אם מוסיפים תשובה חלקית להנחיה, המודל מתחיל לפעול לפי תבנית או פורמט רצויים.

פירוק ההנחיות לרכיבים פשוטים

בתרחישי שימוש שבהם נדרשות הנחיות מורכבות, אפשר לעזור למודל להתמודד עם המורכבות הזו על ידי פירוט הדברים לרכיבים פשוטים יותר.

פירוט ההוראות

במקום להוסיף הרבה הוראות להנחיה אחת, כדאי ליצור הנחיה אחת לכל הוראה. אפשר לבחור איזו הנחיה לעבד על סמך הקלט של המשתמש.

הנחיות בשרשור

למשימות מורכבות שכוללות כמה שלבים רצופים, אפשר להפוך כל שלב להנחיה ולקשר את ההנחיות ברצף. בשרשרת הרציפה הזו של הנחיות, הפלט של הנחיה אחת ברצף הופך לקלט של ההנחיה הבאה. הפלט של ההנחיה האחרונה ברצף הוא הפלט הסופי.

תשובות מצטברות

כדאי להשתמש בצבירה כשרוצים לבצע משימות מקבילות שונות בחלקים שונים של הנתונים ולצבור את התוצאות כדי ליצור את הפלט הסופי. לדוגמה, אפשר להורות למודל לבצע פעולה אחת על החלק הראשון של הנתונים, לבצע פעולה אחרת על שאר הנתונים ולצבור את התוצאות.

סיכום

  • כדאי לפרק הוראות מורכבות להנחיה לכל הוראה ולהחליט איזו הנחיה להחיל על סמך הקלט של המשתמש.
  • כדאי לפרק כמה שלבים רצופים להנחיות נפרדות ולקשר אותן כך שהפלט של ההנחיה הקודמת יהפוך לקלט של ההנחיה הבאה.
  • פירוט המשימות המקבילות וצבירה של התשובות כדי ליצור את הפלט הסופי.

ניסוי עם ערכים שונים של פרמטרים

כל קריאה ששולחים למודל כוללת ערכי פרמטרים ששולטים באופן שבו המודל יוצר תגובה. המודל יכול ליצור תוצאות שונות עבור ערכי פרמטרים שונים. כדאי לנסות ערכי פרמטרים שונים כדי למצוא את הערכים הטובים ביותר למשימה. הפרמטרים שזמינים למודלים שונים עשויים להיות שונים. הפרמטרים הנפוצים ביותר הם:

  • אסימוני פלט מקסימלי
  • טמפרטורה
  • למעלה מ-K
  • Top-P

מספר מקסימלי של אסימוני פלט

המספר המקסימלי של אסימונים שאפשר ליצור בתגובה. אסימון הוא ארבעה תווים בערך. 100 אסימונים מקבילים לכ-20 מילים.

מציינים ערך נמוך יותר לתשובות קצרות וערך גבוה יותר לתשובות ארוכות יותר.

טמפרטורה

הטמפרטורה משמשת לדגימה במהלך יצירת התגובה, שמתרחשת כשמחילים את topP ואת topK. הטמפרטורה שולטת במידת האקראיות בבחירת האסימון. טמפרטורות נמוכות טובות להנחיות עם ציפייה לתשובה deterministית יותר, ולא לתשובה פתוחה או יצירתית, ואילו טמפרטורות גבוהות יכולות להוביל לתוצאות יותר מגוונות או יצירתיות. טמפרטורה של 0 היא גורמית, כלומר תמיד נבחרת התשובה עם ההסתברות הגבוהה ביותר.

ברוב התרחישים לדוגמה, כדאי להתחיל בטמפרטורה של 0.2. אם התשובה שמוחזרת על ידי המודל קצרה מדי, גנרית מדי או שהיא תשובה חלופית, כדאי לנסות להגדיל את הטמפרטורה.

למעלה מ-K

האפשרות Top-K משנה את האופן שבו המודל בוחר אסימונים לפלט. כשה-top-K הוא 1, המשמעות היא שהאסימון הבא שנבחר הוא בעל ההסתברות הגבוהה ביותר מבין כל האסימונים במילון של המודל (נקרא גם פענוח חמדן). לעומת זאת, כשה-top-K הוא 3, המשמעות היא שהאסימון הבא נבחר מתוך שלושת האסימונים עם ההסתברות הגבוהה ביותר באמצעות טמפרטורה.

בכל שלב של בחירת אסימונים, מתבצעת דגימה של 'ה-K הטובים ביותר' עם ההסתברויות הגבוהות ביותר. לאחר מכן, האסימונים מסוננים עוד יותר על סמך top-P, והאסימון הסופי נבחר באמצעות דגימת טמפרטורה.

מציינים ערך נמוך יותר כדי לקבל פחות תשובות אקראיות וערך גבוה יותר כדי לקבל יותר תשובות אקראיות. ערך ברירת המחדל של top-K הוא 40.

Top-P

ההיבט העליון (P) משנה את האופן שבו המודל בוחר אסימונים לפלט. האסימונים נבחרים מהסיכוי הגבוה ביותר (ראו top-K) עד הנמוך ביותר, עד שסכום הסיכויים שלהם שווה לערך top-P. לדוגמה, אם ההסתברות של האסימונים A,‏ B ו-C היא 0.3,‏ 0.2 ו-0.1, וערך ה-top-P הוא 0.5, המודל יבחר את האסימון הבא מתוך A או B באמצעות הטמפרטורה, ויחרוג את C מהאפשרויות.

מציינים ערך נמוך יותר כדי לקבל פחות תשובות אקראיות וערך גבוה יותר כדי לקבל יותר תשובות אקראיות. ערך ברירת המחדל העליון (P) הוא 0.95.

עיצוב הנחיות הוא תהליך איטרטיבי, ולרוב נדרשות כמה חזרות לפני שמקבלים את התשובה הרצויה באופן עקבי. בקטע הזה מפורטות כמה פעולות שאפשר לנסות כשמשנים את ההנחיות.
