Hướng dẫn: Làm quen với API Gemini

Sử dụng bộ sưu tập để sắp xếp ngăn nắp các trang Lưu và phân loại nội dung dựa trên lựa chọn ưu tiên của bạn.


Xem trên ai.google.dev Chạy trong Google Colab Xem nguồn trên GitHub

Nếu muốn nhanh chóng dùng thử Gemini API, bạn có thể dùng các lệnh curl để gọi các phương thức trong API REST. Ví dụ trong trường hợp này hướng dẫn hiển thị lệnh gọi cho từng phương thức API.

Chiến lược phát hành đĩa đơn Colab dùng mã Python để đặt biến môi trường và hiển thị hình ảnh, nhưng bạn không cần Colab hoạt động với API REST. Bạn sẽ có thể chạy tất cả các ví dụ về curl bên ngoài Colab mà không cần sửa đổi, miễn là bạn có Đã thiết lập API_KEY như được mô tả trong phần tiếp theo.

Đối với mỗi lệnh curl, bạn phải chỉ định tên mô hình thích hợp và API của mình .

Thiết lập khoá API

Để sử dụng Gemini API, bạn sẽ cần có khoá API. Tạo một khoá trong Google AI Studio nếu bạn chưa có.

Tải khoá API

Trong Colab, hãy thêm khoá vào trình quản lý bí mật trong phần "🔑" trong bảng điều khiển bên trái. Đặt tên cho tệp đó là GOOGLE_API_KEY. Sau đó, bạn có thể thêm biến này làm biến môi trường để truyền khoá vào lệnh gọi curl.

Trong thiết bị đầu cuối, bạn chỉ cần chạy GOOGLE_API_KEY="Your API Key".

import os
from google.colab import userdata

os.environ['GOOGLE_API_KEY'] = userdata.get('GOOGLE_API_KEY')

Nhập văn bản và hình ảnh

Gemini cung cấp nhiều mô hình có thể xử lý dữ liệu nhập đa phương thức (Các mô hình Gemini 1.5) để bạn có thể nhập cả hai văn bản và hình ảnh. Hãy nhớ xem lại các yêu cầu về hình ảnh đối với câu lệnh.

Khi câu lệnh nhập bao gồm cả văn bản và hình ảnh, hãy sử dụng mô hình Gemini 1.5.

Các đoạn mã sau đây giúp bạn tạo một yêu cầu và gửi yêu cầu đó đến API REST.

Cuộc trò chuyện nhiều lượt (trò chuyện)

Khi sử dụng Gemini, bạn có thể tạo các cuộc trò chuyện dạng tự do ở nhiều chặng.

curl https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=$GOOGLE_API_KEY \
    -H 'Content-Type: application/json' \
    -X POST \
    -d '{
      "contents": [
        {"role":"user",
         "parts":[{
           "text": "Write the first line of a story about a magic backpack."}]},
        {"role": "model",
         "parts":[{
           "text": "In the bustling city of Meadow brook, lived a young girl named Sophie. She was a bright and curious soul with an imaginative mind."}]},

Cấu hình

Mỗi câu lệnh bạn gửi đến mô hình đều có các giá trị thông số kiểm soát cách mô hình tạo phản hồi. Mô hình có thể tạo ra kết quả khác nhau cho các giá trị thông số khác nhau. Tìm hiểu thêm về các thông số của mô hình.

Ngoài ra, bạn có thể sử dụng các chế độ cài đặt an toàn để điều chỉnh khả năng nhận được những câu trả lời có thể được coi là gây hại. Theo mặc định, chế độ cài đặt an toàn sẽ chặn nội dung có xác suất trung bình và/hoặc cao là nội dung không an toàn trên tất cả phương diện. Tìm hiểu thêm về chế độ cài đặt an toàn.

Ví dụ sau đây chỉ định giá trị cho tất cả các tham số của phương thức generateContent.

Nhúng

Nhúng là một kỹ thuật dùng để biểu diễn thông tin dưới dạng danh sách các số có dấu phẩy động trong một mảng. Với Gemini, bạn có thể trình bày văn bản (từ, câu và khối văn bản) ở dạng vectơ, giúp bạn dễ dàng so sánh và đối chiếu các mục nhúng. Ví dụ: hai văn bản có cùng chủ đề hoặc một tình cảm phải có các liên kết nhúng tương tự nhau. Các nội dung này có thể được xác định thông qua các kỹ thuật so sánh toán học như tính tương đồng về cosin.

Thông tin về mô hình

Tải mô hình

Nếu bạn GET URL của một mô hình, API sẽ sử dụng phương thức get để trả về thông tin về mô hình đó, chẳng hạn như phiên bản, tên hiển thị, giới hạn về mã thông báo đầu vào, v.v.

curl https://generativelanguage.googleapis.com/v1beta/models/gemini-pro?key=$GOOGLE_API_KEY
